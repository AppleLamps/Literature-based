# REPORT.md - Literature-Based Discovery run

_Status: build complete; validation gate in progress; no live discovery run has
been executed yet. This file is regenerated by `python -m lbd run`._

## What exists right now

- A complete, documented, reusable LBD pipeline (`lbd/`) with a one-command CLI.
- The Swanson **validation gate** has been run on the Raynaud case and is mid-tuning
  (see VALIDATION.md): under a pre-1986 freeze the engine recovers
  Eicosapentaenoic Acid at rank 227/3520 with the correct bridges (Blood Viscosity,
  Platelet Aggregation, Blood Platelets). The Migraine case has not finished.
- **No `lbd run` discovery on live A-terms has been executed**, so there are no
  vetted hypotheses to report here yet, and `hypotheses.csv` / `results/briefs/`
  have not been produced. Running discovery before the gate fully passes would be
  premature by the project's own rule ("no credit for new hypotheses until the
  known ones come back").

## Next steps to a finished run

1. Finish closing the ranking gap so both Swanson targets land in the top-N under
   the freeze (VALIDATION.md, open item). Both cases are cached, so this is fast.
2. Run discovery on three large-literature A-terms:
   ```
   .venv/bin/python -m lbd run --a-term "Alzheimer Disease" \
       --a-term "Psoriasis" --a-term "Parkinson Disease"
   ```
3. This populates `hypotheses.csv`, per-survivor briefs in `results/briefs/`, and
   regenerates this report with corpus sizes, candidate/survivor counts, and
   per-survivor confidence.

## Standing caveats (apply to all output once produced)

- These will be **hypotheses generated from literature structure, not findings**.
  Each survivor is a currently-unstudied A--C link bridged by shared intermediates;
  none is experimentally tested here.
- The MeSH co-occurrence substrate is undirected, so the *sign* of each mechanistic
  edge is not verified; the mechanistic-coherence gate is a warning, not a pass, for
  every candidate (see DATA.md). A domain expert must confirm directionality.
- A "gap" means absence of co-occurrence / preprint evidence in the indexed
  literature, not proof the link is novel or correct.

> Every hypothesis this pipeline produces requires expert appraisal and
> experimental validation before it is anything more than a lead.
